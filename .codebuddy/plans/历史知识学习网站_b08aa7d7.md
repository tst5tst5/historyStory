## 产品概述

历史知识学习网站（Web端），用户通过地理区域和时间轴双维度浏览世界历史知识，支持响应式横竖屏布局。

## 核心功能

- **登录系统**：支持手机号/邮箱登录，开发阶段写死测试账号（手机号 13800138000 / 邮箱 learner@history.com，统一密码 history2024）
- **地理区域筛选（按需添加模式）**：默认只显示亚洲-中国；通过弹窗添加大洲/国家（弹窗内支持搜索筛选、大洲全选、国家单选，已添加国家标记✓不可重复添加）；国家可删除（✕按钮），删除后隐藏其历史事件；大洲下所有国家删除后该大洲自动隐藏
- **时间轴展示**：横向年份时间轴，支持拖拽滚动条和手型鼠标拖动，显示年份+事件概要，hover显示事件详情弹窗
- **横竖屏响应式**：横屏时大洲国家面板在左侧，竖屏时在顶部，支持手动切换
- **顶部固定工具栏**：Logo+站名"史鉴"、**添加区域按钮（核心功能，金色醒目样式）**、搜索框、横竖屏切换、收藏夹、暗黑模式、用户头像+退出登录

## Tech Stack

- **前端框架**：React 18 + TypeScript
- **构建工具**：Vite 5
- **样式方案**：Tailwind CSS 3.4.17 + tailwind-merge + tailwindcss-animate
- **图标库**：Lucide React + react-icons
- **PostCSS**：postcss 8.5 + autoprefixer ^10.4.20
- **数据存储**：前端内存状态 + LocalStorage（收藏、主题偏好、已选区域）
- **认证**：前端硬编码账号，Context状态管理

## Implementation Approach

### 整体策略

使用Vite脚手架初始化React+TS项目，全局状态通过React Context管理（认证、主题、区域选择），核心交互为自定义横向时间轴组件和按需添加的区域选择弹窗。响应式布局通过CSS媒体查询 + window.matchMedia检测横竖屏方向。

### 关键技术决策

1. **区域选择采用"按需添加"模式**：默认仅显示亚洲-中国，用户通过工具栏醒目的"添加区域"按钮打开弹窗按需添加，避免一次性展示全部大洲国家导致面板过长。弹窗内左右分栏（大洲列表|国家列表），顶部搜索框支持按名称筛选，已添加国家显示✓标记
2. **时间轴拖拽**：自定义实现（mousedown/mousemove/mouseup + touch事件），避免引入重依赖，更灵活控制样式和交互
3. **区域状态管理**：RegionContext维护已选区域列表（continent[] 每个含已选country[]），删除国家时同步移除其事件，大洲下无国家时自动移除大洲
4. **横竖屏检测**：`window.matchMedia('(orientation: landscape)')` + 手动切换按钮覆盖，CSS Grid布局切换
5. **主题切换**：Tailwind CSS `dark:` 变体 + localStorage持久化

### 性能考虑

- 历史事件列表可能较长，时间轴使用虚拟化渲染（仅渲染可视区域附近的事件节点）
- 区域筛选变化时使用debounce避免频繁重渲染
- 事件详情hover使用CSS transform动画，避免重排

## Implementation Notes

- 时间轴拖拽需同时处理鼠标事件和touch事件，touch事件需`preventDefault`避免页面滚动
- 公元前年份用负数存储，显示时加"公元前"前缀
- 登录状态持久化到sessionStorage，关闭标签页后需重新登录
- 收藏数据持久化到localStorage，键名`history_bookmarks`
- 已选区域持久化到localStorage，键名`history_selected_regions`，刷新页面保留选择
- RegionContext删除国家时需同步过滤时间轴事件，大洲下无国家时自动从列表移除
- AddRegionModal中搜索需同时匹配大洲名和国家名，支持中文和英文
- RegionTag的✕删除按钮hover时变红提示，删除无需二次确认（可重新添加）
- tsconfig.app.json需设置`verbatimModuleSyntax: false`、`noUnusedLocals: false`、`noUnusedParameters: false`
- vite.config.ts需设置`server.host: '0.0.0.0'`和`server.allowedHosts: true`
