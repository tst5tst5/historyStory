## 产品概述

历史知识学习网站（Web端），用户通过地理区域和时间轴双维度浏览世界历史知识，支持响应式横竖屏布局。

## 核心功能

- **登录系统**：支持手机号/邮箱登录，开发阶段写死测试账号（手机号 13800138000 / 邮箱 learner@history.com，统一密码 history2024）
- **地理区域筛选**：大洲→国家级联选择，选中大洲自动全选其下所有国家，支持单独选/取消单个国家
- **时间轴展示**：横向年份时间轴，支持拖拽滚动条和手型鼠标拖动，显示年份+事件概要，hover显示事件详情弹窗
- **横竖屏响应式**：横屏时大洲国家面板在左侧，竖屏时在顶部，支持手动切换
- **顶部固定工具栏**：Logo+站名、搜索框、横竖屏切换、收藏夹、暗黑模式、用户头像+退出登录

## 补充细节

- 历史事件数据使用公元年份制，内置JSON数据文件
- 时间轴需处理公元前/公元后的标记
- 拖拽需同时支持鼠标拖动和触摸滑动
- 大洲至少覆盖：亚洲、欧洲、非洲、北美洲、南美洲、大洋洲，每洲含代表性国家
- hover详情卡片需有进入/退出动画

## 技术栈

- **前端框架**：React 18 + TypeScript
- **构建工具**：Vite
- **样式方案**：Tailwind CSS 3
- **图标库**：Lucide React
- **数据存储**：前端内存状态 + LocalStorage（收藏、主题偏好）
- **认证**：前端硬编码账号，Context状态管理

## 实现方案

### 整体策略

使用Vite脚手架初始化React+TS项目，全局状态通过React Context管理（认证、主题、区域选择），核心交互为自定义横向时间轴组件（支持鼠标拖拽+触摸滑动+滚动条），响应式布局通过CSS媒体查询 + window.matchMedia检测横竖屏方向。

### 关键技术决策

1. **时间轴拖拽**：不使用第三方时间轴库，自定义实现（mousedown/mousemove/mouseup + touch事件），避免引入重依赖，更灵活控制样式和交互
2. **区域选择数据结构**：树形结构 `大洲 → 国家[]`，使用递归checkbox逻辑实现全选/半选/单选
3. **横竖屏检测**：`window.matchMedia('(orientation: landscape)')` + 手动切换按钮覆盖，CSS Grid布局切换
4. **主题切换**：Tailwind CSS `dark:` 变体 + localStorage持久化

### 性能考虑

- 历史事件列表可能较长，时间轴使用虚拟化渲染（仅渲染可视区域附近的事件节点）
- 区域筛选变化时使用debounce避免频繁重渲染
- 事件详情hover使用CSS transform动画，避免重排

## 实现备注

- 时间轴拖拽需同时处理鼠标事件和touch事件，touch事件需`preventDefault`避免页面滚动
- 公元前年份用负数存储，显示时加"公元前"前缀
- 登录状态持久化到sessionStorage，关闭标签页后需重新登录
- 收藏数据持久化到localStorage，键名`history_bookmarks`

## 设计风格

采用现代深色学术风格（Dark Academia），融合Glassmorphism元素，营造历史学习的沉浸氛围。整体色调偏暖，以深棕/金色为主调，搭配纸张质感的浅色内容区域。
